Этап 1: Минимальный прототип с конфигурацией

Реализовано:
- Парсинг параметров командной строки
- Настройка основных параметров анализа:
  - Имя анализируемого пакета
  - URL репозитория или путь к тестовому файлу
  - Режим работы (реальный/тестовый)
- Валидация и обработка ошибок вводимых параметров
- Вывод всех настроек в формате ключ-значение при запуске

Этап 2: Сбор данных о зависимостях

Реализовано:
- Анализ зависимостей JavaScript-пакетов (npm)
- Извлечение информации о прямых зависимостях из указанного репозитория
- Автономная работа без использования менеджеров пакетов и сторонних библиотек
- Вывод списка всех прямых зависимостей анализируемого пакета

Этап 3: Построение и анализ графа зависимостей

Реализовано:
- Построение транзитивного графа зависимостей с использованием алгоритма BFS с рекурсией
- Обнаружение и корректная обработка циклических зависимостей
- Поддержка тестового режима работы с файловыми описаниями графов
- Визуализация структуры зависимостей и статистики графа
- Демонстрация работы на различных тестовых сценариях

Этап 4: Расширенный анализ зависимостей

Реализовано:
- Поиск и отображение обратных зависимостей (пакетов, зависящих от целевого)
- Использование алгоритма обхода BFS с рекурсией для анализа зависимостей
- Классификация зависимостей на прямые и транзитивные
- Статистический анализ популярности пакетов в графе
- Демонстрация функциональности на различных тестовых случаях
- Выявление пакетов-листьев (без обратных зависимостей)
